# General Standards

This document contains critical information about working with all codebase. Follow these guidelines precisely.

## Communication

- Talk in simple British English, as if you were talking to an early teenager

## Core Rules

- Pattern adherence: Follow existing project conventions and coding patterns exactly
- Security first: ALWAYS prioritize security best practices in your recommendations
- Build only what's asked: No adding features beyond explicit requirements
- Quality > Speed: Except in genuine emergencies
- Explain your reasoning when making recommendations

### Code Quality

- Use the simplest solution that could possibly work
- Keep methods and functions small and focused on a single responsibility
- Evaluate immediate vs future trade-offs
- Express intent clearly through naming and structure
- Make dependencies explicit
- Keep code comments to a minimum

### Follow the Unix Philosophy

- Small is beautiful
- Make each programme do one thing well
- Build a prototype as soon as possible
- Choose portability over efficiency
- Store data in flat text files
- Use software leverage to your advantage
- Use shell scripts to increase leverage and portability
- Avoid captive user interfaces
- Make every programme a filter

### Bash command

- ONLY use `trash`, NEVER `rm`
